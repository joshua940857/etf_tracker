🎯 系統簡介

這是一個自動化的投資分析系統，結合了 Momentum 交易策略和 ETF 高低點追蹤。 系統每週自動執行，將結果展示在這個 Web 儀表板上。

⚙️ 核心功能

📈 Momentum 追蹤

監控投資組合的動量表現，識別買賣信號

🎯 ETF 高低點

追蹤 VT、VOO、VTI 的高低點和距離百分比

🔄 自動執行

每週定時自動分析，無需手動干預

📊 可視化展示

生成圖表和報告，易於理解和分析

📅 執行計畫

⏰ 定期執行

執行時間： 每週一至五 09:30 UTC
台北時間： 每週一至五 17:30
自動更新： GitHub Actions 提供動力

📊 數據來源

  • ETF 價格數據： Yahoo Finance（通過 yfinance 庫）
  • Momentum 數據： 內部系統計算
  • 更新頻率： 每周一次（或手動觸發）

🔍 追蹤指標說明

ETF 高低點

  • 高點 (H)： 指定時間窗口內的最高價格
  • 低點 (L)： 指定時間窗口內的最低價格
  • 距離高點 (%)： 當前價格相對高點的百分比變化
    • 💚 綠色（正值）：價格高於高點（罕見）
    • ❌ 紅色（負值）：價格低於高點（正常）
  • 距離低點 (%)： 當前價格相對低點的百分比變化
    • 💚 綠色（>5%）：距低點較遠，可能被高估
    • 🟡 黃色（2-5%）：中等距離
    • ❌ 紅色（<2%）：接近低點，可能被低估

📋 時間窗口

系統追蹤以下時間窗口：

  • 1M： 最近 1 個月（約 21 個交易日）
  • 3M： 最近 3 個月（約 63 個交易日）
  • 6M： 最近 6 個月（約 126 個交易日）
  • 1Y： 最近 1 年（約 252 個交易日）
  • YTD： 年初至今
  • 3Y： 近 3 年全期

❓ 常見問題

Q: 為什麼某些 ETF 的數據不完整？

A: 如果某個時間窗口的數據尚未完全形成（如新年的 YTD），系統會跳過該部分。

Q: 數據有多及時？

A: 數據在美股收盤後更新（UTC 時間 21:00 = 台北時間隔日 05:00）。系統於 UTC 09:30 執行，使用前一交易日的收盤數據。

Q: 能否改變執行時間？

A: 可以，需要修改 GitHub Actions 工作流的 cron 時間設置。詳見 GitHub 倉庫的設置。

Q: 如何獲得通知？

A: 可以設置多種通知方式：

  • GitHub 通知（每次執行完成）
  • 郵件通知（可選配置）
  • Slack/Discord 通知（可選配置）
